AT&T Drive connected vehicle cloud provides a way for automotive manufacturers to deliver cloud-based infotainment features to vehicles and provides tight integration between apps and services.

This accelerator toolkit is built to offer app developers a fast way to build and deploy AngularJS/HTML5 based apps to AT&T Drive CVC and contains these resources:

  • Sample apps for quick start
  • Common UI Kit API for rapid development
  • Vehicle Data and Event Controller API
  • Packaging and deployment to app marketplace

Amongst the components of this toolkit is a head unit simulator that will allow you to test and debug your app as you build it. Some assembly is required and the instruction are included here, just follow the environment setup instructions.

Once you have the head unit simulator running you should pull the code for example apps from the repository to begin writing your first app.

Previewing Your App

  1. Run head unit simulator then open Settings > Developer and enter your app IP or url. Make sure your app is pointing to SPAN IP at 127.0.0.1 and the port matches the port listed in settings.
  2. Enable Developer Mode and open the tray.
  3. You will see your app icon.
  4. Tap to launch you app.

AT&T Drive supports a common AngularJS based application framework for app development. The objective is to allow Automotive OEM to optimize the experience of all apps and services effortlessly in one single location.

UI Kit contains the UI elements with very basic styling and interaction, and when used the apps will provide a unified experience across the app ecosystem.

Apps developed for AT&T Drive CVC should all contain the common structure that is already built-in into the sample apps. App menu, routes, and two-view example is available to speed up the development time.

Head Unit JavaScript API

Head Unit apps built for AT&T Drive communicate with other apps and services via Data and Event Controller (DEC). Sample apps contain the necessary libraries required to interface with services like navigation or vehicle itself.

DEC will also maintain the user settings and app preferences in the cloud as well as identity management so that the user remains authenticated with same credentials across all apps and services in the vehicle.

DEC JavaScript API is evolving to include upcoming features and enhancements. For full documentation see Vehicle API.

Web API

Vehicle services are exposed outside of the Head Unit via REST API. Apps can be built on external devices to control the vehicle and subscribe to data and events. Additionally, External apps also communicate with the Head Unit apps.

Example of this integration may be an external “engine start” app, or pushing an address to the Head Unit navigation app.
